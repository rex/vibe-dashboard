# Vibe Dashboard

**Mission control for vibe coding.** A native macOS app that watches a fleet of
agentic repositories under `~/Code` — and the AI coding agents editing them — in
real time, and tells you the truth about their health.

Vibe Dashboard is the realtime, multi-repo face of an in-chat "repo checkup": it
continuously scores each repo against its own `VIBE.yaml` policy and against the
things only a local app can see — live coding-agent sessions, git worktree
sprawl, doc bloat, changelog staleness, hygiene problems, lifecycle hooks, and
MCP servers. When an agent spins up a multi-agent workflow, you can **watch every
lane stream at once**, following each agent through phase hand-offs without
clicking around.

It is built to one principle above all: **near-zero false positives.** An
oversight tool you can't trust is worse than no tool — so every number is
measured or honestly labelled, never fabricated and never a guess dressed up as
a fact.

---

## Highlights

- **Fleet health at a glance** — every managed repo scored on a single
  severity-weighted compliance grade, with a **"why this grade"** breakdown that
  lists each deduction. The factor list *is* the grade; they can never disagree.
- **Live agent & workflow watching** — detects running Claude Code / Codex
  sessions from their transcripts (not `ps` guessing) and opens a resizable watch
  window. Multi-agent workflows render as **concurrency lanes** whose streams
  continue across phase hand-offs behind a loud in-stream "hop" divider —
  markdown-rendered, tool calls collapsed-but-expandable, sticky bottom-follow.
- **Push-based, not poll-based** — FSEvents drives instant fleet-card updates and
  ~150 ms watch-pane ticks; per-repo re-scores fire on real filesystem changes,
  noise-filtered and echo-cooled. Idle CPU stays flat (indicators *draw*; they
  never animate layout).
- **Honest hygiene detection** — merge-conflict markers (line-anchored), tracked
  secrets (only when a file's *contents* actually leak credentials), committed
  dependency dirs, forgotten stashes, god-files, doc bloat, skeleton drift —
  tuned hard against alert fatigue.
- **Waivers that mean something** — waive a finding and its weight leaves the
  grade instantly (with disclosure and auto-expiry), not just the feed.
- **Real actions** — exclude a file from architecture scope, record skill
  provenance, group findings by codebase, and run `make` targets straight into an
  in-app console.
- **Auto-updates** via [Sparkle](https://sparkle-project.org): signed appcast,
  EdDSA + notarization gated, auto-check + prompt.

## Design

Dark, mono-dominant, one acid-lime accent that doubles as the "healthy / live"
signal. Two vendored typefaces (JetBrains Mono + Space Grotesk), a Lucide→SF
Symbol bridge, and a single design-token system: every colour, font, radius, and
spacing value flows through `Shared/Theme` (`Theme.color/font/spacing/…`) — no
raw hex or ad-hoc padding in views.

## Requirements

- macOS 14 (Sonoma) or later
- Xcode 16+ with the macOS SDK
- [`xcodegen`](https://github.com/yonaskolb/XcodeGen) — `brew install xcodegen`

## Build & run

```bash
brew install xcodegen     # one-time
make run                  # generate project, build, launch
make test                 # unit tests (Swift Testing)
make validate             # build · test · lint · architecture · docs · audit — the gate
```

`project.yml` is the source of truth; the `.xcodeproj` is regenerated by
xcodegen and git-ignored. Never hand-edit the project file.

> **Not sandboxed, on purpose.** Vibe reads arbitrary repos under `~/Code` and
> shells out to `git` / `make`, which the App Sandbox forbids. It is instead
> distributed directly as a **notarized, hardened-runtime** app — never through
> the Mac App Store. See [docs/RELEASE.md](docs/RELEASE.md).

## How it works

```
Shared/
  Scan/        discovery → per-repo probing (git, census, docs, hooks, mcp,
               policy, hygiene) → agent-session detection → derive grades.
               Off the main actor; Sendable-clean under Swift 6 strict concurrency.
  Services/    FleetStore — the single @MainActor @Observable UI source of truth.
  Models/      Sendable value types (Repo, Finding, AgentInfo, GradeFactor…).
  Theme/ DesignSystem/   design tokens + component library + brand marks.
VibeDashboard/
  Chrome/      toolbar, sidebar, inspector, console, menus, updater.
  Views/       Fleet, Agents, Findings, Skills, Autopilot, 7-tab Repo detail.
  Watch/       the per-agent / per-workflow transcript watch window.
```

Each repo declares policy in a **`VIBE.yaml`** (stack, architecture limits,
required gates, skills). The scanner keeps only managed repos — those carrying a
`VIBE.yaml` / `AGENTS.md` / workspace marker — and grades each against its
policy. Files stay under a hard 400-line / soft 250-line limit: the app enforces
the same limit it measures.

Grading is **severity-weighted**: `compliance = 100 − Σ deductions`, health from
score bands, and critical factors (unmanaged repo, unparseable policy, merge
markers, tracked secrets, an unguarded live agent) force a danger grade.
