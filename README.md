# Vibe Dashboard

> Mission control for vibe coding. `vibe‚ñÆ`

A native **macOS** app that keeps a fleet of [agentic-skeleton] repos ‚Äî and the
autonomous coding agents editing them ‚Äî in policy and in sight. It reads
`~/Code` directly off disk: git state, worktree sprawl, live agent sessions,
doc bloat, Serena projects, `.claude`/`.codex`/`.cursor` lifecycle hooks, MCP
servers, and each repo's `VIBE.yaml` contract ‚Äî then renders it all as a dark,
mono-dominant instrument panel.

Where the in-chat `agentic-checkup` skill runs a one-shot health report on a
single repo, Vibe runs **continuously, across the whole fleet**, and gives a
local app the powers a remote worker never had: seeing an agent editing a repo
*right now*, on what branch, touching how many files.

## Quick start

```bash
brew install xcodegen          # one-time
make build-mac                 # xcodegen generate + xcodebuild (unsigned)
make run                       # build + launch
open VibeDashboard.xcodeproj   # or work in Xcode
```

## Architecture

- **`Shared/`** ‚Äî the portable core (no AppKit UI):
  - `Theme/` ‚Äî the design-system chokepoint (tokens ported from the Vibe
    Dashboard design system: ink ramp, acid-lime accent, JetBrains-Mono-first
    type, tight radii, phosphor glow).
  - `DesignSystem/` ‚Äî SwiftUI primitives (panel, button, tag, status badge,
    severity tag, meter, gate row, tabs) + brand marks.
  - `Models/` ‚Äî `Sendable` value types (`Repo`, `Fleet`, `Gate`, `Finding`, ‚Ä¶).
  - `Scan/` ‚Äî the real `~/Code` scanner + git / census / docs / serena / hooks /
    mcp / policy probes and a `Process` runner.
  - `Services/` ‚Äî `FleetStore` (`@Observable @MainActor`), settings, formatters.
- **`VibeDashboard/`** ‚Äî the macOS app: window chrome (toolbar, source list,
  inspector, console, status bar, menu bar + menu-bar extra), the screens
  (Fleet, Agents, Findings, Skills, Autopilot, Repo detail, Workspace), and
  overlays (command palette, sheets, toasts, tweaks).

The `.xcodeproj` is a **build artifact** ‚Äî gitignored, regenerated by
`xcodegen generate` from `project.yml` (the source of truth). Never hand-edit it.

## Not sandboxed ‚Äî on purpose

Vibe is a local developer tool: it reads arbitrary repos and shells out to
`git` / `make`. That is incompatible with the App Sandbox, so the app runs
un-sandboxed and is distributed directly (notarized), never via the Mac App
Store. See `VibeDashboard/VibeDashboard.entitlements`.

## Versioning

`MARKETING_VERSION` = `MAJOR.(MINOR_BASE + commits-since-VERSION-last-touched)`
from `VERSION`; build number = `git rev-list --count HEAD`. Both are injected at
build time by `Scripts/generate-build-info.sh` ‚Üí `Generated/BuildInfo.swift`.
